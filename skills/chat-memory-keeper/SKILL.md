# Chat Memory Keeper - 群聊记忆管家

为 OpenClaw IM 机器人设计的记忆管理技能，从日常群聊中自动提取关键信息，构建人物档案和备忘录。

## 核心定位

| 维度 | 定位 |
|------|------|
| **适用场景** | 钉钉/微信/Telegram 等群聊机器人 |
| **核心功能** | 信息提取 + 人物档案 + 备忘录 |
| **触发方式** | 被动监听 + 主动整理 |
| **隐私保护** | 本地存储，可配置脱敏 |

---

## 信息提取能力

### 提取要素

| 要素 | 说明 | 示例 |
|------|------|------|
| **📅 时间** | 消息时间、事件时间 | "明天下午 3 点"、"下周一" |
| **👤 人物** | 提及的人名、角色 | "张三"、"产品经理"、"老板" |
| **📍 地点** | 地理位置、场所 | "北京"、"公司会议室"、"星巴克" |
| **🎯 事件** | 计划、任务、约定 | "下周上线"、"周五聚餐" |
| **💭 心情** | 情绪状态、感受 | "好累"、"太开心了"、"压力大" |
| **🔖 主题** | 讨论话题、项目 | "双 11 活动"、"招聘计划" |

### 提取示例

```
原始消息:
"下周五 (3 月 14 日) 下午 3 点，张三和李四在北京办公室开会，
讨论双 11 活动方案。王五说最近压力好大，需要招人。"

提取结果:
{
  "timestamp": "2026-03-06 14:30:00",
  "events": [
    {
      "type": "meeting",
      "time": "2026-03-14 15:00:00",
      "location": "北京办公室",
      "participants": ["张三", "李四"],
      "topic": "双 11 活动方案"
    }
  ],
  "people": [
    {
      "name": "王五",
      "mood": "压力大",
      "context": "需要招人"
    }
  ],
  "topics": ["双 11 活动", "招聘"]
}
```

## 使用方式

### 被动监听模式

机器人自动监听群聊，静默提取信息：

```
[群聊消息]
张三：下周五下午 3 点开会啊
李四：好的，在哪里？
张三：北京办公室会议室

[机器人后台处理]
→ 提取事件：会议
→ 更新时间：下周五 15:00
→ 更新地点：北京办公室会议室
→ 更新参与人：张三、李四
→ 保存到备忘录
```

### 主动整理模式

用户触发整理命令：

```
用户：@机器人 整理今天的聊天

机器人回复:
📋 今日聊天摘要 (2026-03-06)

📅 事件：2 个
  • 会议：下周五 15:00，北京办公室
  • 聚餐：周日 18:00，海底捞

💭 心情：3 人提及
  • 王五：压力大
  • 张三：期待
  • 李四：开心

📝 待办：3 项
  • 张三：准备方案 (3/10)
  • 李四：联系场地 (3/12)
  • 王五：发布招聘 (3/8)

详情查看：memory/memos/2026-03-06.md
```

### 查询命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `@机器人 查 {人名}` | 查看人物档案 | `@机器人 查 张三` |
| `@机器人 查 {日期}` | 查看备忘录 | `@机器人 查 3 月 6 日` |
| `@机器人 查 待办` | 查看所有待办 | `@机器人 查 待办` |
| `@机器人 查 事件` | 查看近期事件 | `@机器人 查 事件` |
| `@机器人 导出` | 导出数据 | `@机器人 导出` |

## 配置说明

### 配置文件 (`memory/chat-memory-config.json`)

```json
{
  "enabled": true,
  "groups": [
    {
      "id": "dingtalk-group-123",
      "name": "产品讨论群",
      "enabled": true,
      "extractLevel": "detailed"
    }
  ],
  "extraction": {
    "events": true,
    "moods": true,
    "locations": true,
    "people": true,
    "tasks": true
  },
  "privacy": {
    "maskPhoneNumbers": true,
    "maskIdCards": true,
    "maskBankAccounts": true,
    "retentionDays": 365
  },
  "storage": {
    "profilesPath": "memory/profiles/",
    "memosPath": "memory/memos/",
    "exportFormat": "markdown"
  },
  "notifications": {
    "dailySummary": true,
    "summaryTime": "22:00",
    "upcomingEvents": true,
    "eventReminderHours": 24
  }
}
```

### 配置项说明

| 配置项 | 说明 | 可选值 |
|--------|------|--------|
| `enabled` | 是否启用 | `true`/`false` |
| `groups` | 监听的群列表 | 群 ID 列表 |
| `extractLevel` | 提取详细程度 | `basic`/`standard`/`detailed` |
| `privacy.mask*` | 脱敏设置 | `true`/`false` |
| `retentionDays` | 数据保留天数 | 数字 |
| `notifications.dailySummary` | 每日摘要 | `true`/`false` |
| `notifications.summaryTime` | 摘要时间 | `"22:00"` |

## 隐私保护

### 自动脱敏

```
原始消息:
"我手机号 13800138000，身份证 110101199001011234"

脱敏后存储:
"我手机号 138****8000，身份证 110101****1234"
```

### 数据保留策略

| 数据类型 | 保留期限 | 到期处理 |
|----------|----------|----------|
| 事件记录 | 365 天 | 归档 |
| 心情记录 | 180 天 | 删除 |
| 待办事项 | 完成后 30 天 | 归档 |
| 人物档案 | 永久 | - |
| 聊天记录 | 不存储原始消息 | 仅存提取结果 |

## 最佳实践

### 群聊场景建议

| 场景 | 推荐配置 |
|------|----------|
| 工作群 | 提取事件 + 待办，关闭心情 |
| 朋友群 | 提取心情 + 事件，关闭待办 |
| 家庭群 | 提取事件 + 地点，简化模式 |
| 项目群 | 全量提取，详细模式 |

### 使用技巧

1. **定期整理**: 每周回顾一次备忘录
2. **及时更新**: 发现错误手动修正档案
3. **合理脱敏**: 敏感信息开启自动脱敏
4. **适度使用**: 避免过度记录侵犯隐私

## 故障排除

### 问题 1: 提取不准确

**解决**:
```
用户：@机器人 重新分析刚才的消息
```

### 问题 2: 档案重复

**解决**:
```
用户：@机器人 合并 张三 和 老张 的档案
```

### 问题 3: 隐私担忧

**解决**:
```
用户：@机器人 删除我的所有数据
→ 执行删除并确认
```
